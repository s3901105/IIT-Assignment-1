My Project Idea: Hoop Stats™

Overview

I have decided to develop a smartphone app that tracks the performance and statistics of a player during official basketball games. Though we would assume that the player is also the user of the app, this is not strictly case it could be a parent of the player or a friend operating the software. This app will also be connected to a device that records the player during the game, allowing for cool replays and post-game analysis. Note that official basketball games are games played where there are teams who play in season-round competitions, with referees, team coaches, etc. There are also scorers who oversee putting points on the board when a team score, and they also record different statistics like fouls, layups, free throws, etc. All these statistics are submitted into a database which stores every single player’s data. The app will be able pull the data of the player from that database and display that data in a cleaner and more user-friendly way through the app as well as being able to graph trends in performance. If there is incomplete or inaccurate data, there is the option for the user to manually input data.

Motivation

One of my long-time hobbies is playing basketball, and I even used to play domestic basketball for a couple years. Using my interest in technology I can combine these two interests to create this Basketball stat tracker app. Basketball has always been a beloved sport among Australians and only increases in popularity, according to Basketball Australia as of March 2014 is the second highest team participation sport in Australia. This is especially true in Victoria, where figures released by the Australian Sports Commission's Exercise, Recreation and Sport Survey in 2010 showed that basketball was the most popular sport played in the state of Victoria. My app will allow players to further enhance their game as well as keep records of cool plays and shots taken in-game. In addition, with my extensive knowledge and experience with playing basketball both in a competition and casually, this makes it the perfect project for me.

Description

This software will be called Hoop Stats™ and will be a smartphone app to be used on iOS or android. The app will require some form of internet connection (from mobile data, Wi-Fi, etc.) to back up user data to a cloud storage system or they can plug their smartphone into a physical storage unit through USB. An internet connection is also needed to pull data from the database of players, but if the user opts to, they can input all the data from each game manually. Each user will have to register an account before starting to use the app with basic information like name, date of birth, email, gender, and create a username and password to log in with. After registering, the user will have to find their player profile on whatever database it is stored on. For example, I would find my team W18 GEBC Boys 14C (Winter season 2018 Great Eastern Bees Club boys’ group under 14s C grade) through my club’s web page. Here it will show all the players, and the user will input the link of the profile so the software can start receiving data. The user will also have to give the app permission to access the players data from whatever external database the stats are stored in.

These stats will be used by the app to create graphs showing the performance of the player with different variables over multiple games (e.g. graph showing an increase in 3 point shots taken per game), as well as outline strengths and weaknesses of their games (e.g. User shooting well from the 3 point line, or User is weak at shooting free throws). Even though the stats can be accessed from the organisation provided web page, the Hoop Stats™ will have a cleaner and more organised user interface which the user can navigate through better. The app will allow multiple users to operate on the same smartphone by allowing the current user to log out and log in. Before logging out, the user must sync all their data, whether it is through the internet to a cloud storage or by plugging the smartphone via USB to save it on a physical drive.

Another feature that I would like to add is the ability to use either the phone camera or an external camera to record the games. The user would prop up their phone facing the half of the court they are scoring on, and at half time when the teams swap directions the phone would be propped up on the other side. If they were using an external camera, a Bluetooth/USB function would be required to send the video footage from the camera to the smartphone. While recording there will be three functions: Start Recording, Stop Recording and Save Last 30 Seconds (30 is default, user can choose to change to 15 or 60 seconds). This recording feature will have 2 modes: Whole Match Mode or Action Replay Mode. The Whole Match Mode will record the whole game, and when the Save Last 30 Seconds function is used it will mark those last 30 seconds as a “Key Moment”, so when the user watches the footage after the game, they will have the whole match and moments marked when the function was activated. Action Replay Mode is similar where it records the game, but it only keeps the footage from the Save Last 30 Seconds function. This mode keeps storage requirements low and is recommended for a normal match, whereas the Whole Match Mode is more tailored for important games like the grand final matches or people who have a lot of storage and really wants to analyse their game.
